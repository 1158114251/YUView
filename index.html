Welcome to YUView

YUView is a YUV player with an advanced analytic toolset for Windows, Linux and Mac. It can open and display YUV sequences in a variety YUV formats and supports zooming, easy navigation through the video, side by side view, comparison view, calculation of differences, playlists and more. On this page we will present the key features of YUView. Further information can be found in our wiki.

Getting stated

In order to run YUView you can either get a precompiled binary or you can easily compile YUView from source. We provide precompiled binaries for Windows, Linux and Mac bundeling everything that is needed to run YUView in our releasese section.

If you wish to compile YUView from source you will require to install Qt on your system. The YUView source files are maintained on Github and can be doenloaded here. Please refer to this Wiki page for more information on compiling YUView from source.

YUV View and Navigation

At it's core YUView is a powerful YUV file viewer. This includes various features you are probably familiar with from other media playses like: play, pause, stop and a slider to navigate the sequence. Press the space bar to toggle play/pause, and use the repeat modes (repeat playlist, repeat one sequence, play playlist once).

The center viewer shows the item that is currently selected in the playlist. Press and hold the right or center mouse button to pan the video in the viewer. For a closer look you can zoom into the sequence by selecting the desired area, using the mouse wheel or pressing '+' or '-' on the keyboard.

Playlist

The playlist holds a list of sequences that can be played after each other. Use the open button or just drop supported files here. However, this is not the only thing the playlist can be used for. Are you comparing multiple versions of the same sequence? Use the keyboards 'up' and 'down' arrows in the center view to move the selection in the playlist up and down. Did you start an analysis and you want to work on it later? You don't have to search for all sequences and files that you had opened last time. YUView can save a playlist with all sequences and their settings in a playlist file.

If you want to directly see what the difference between two sequences is, YUView can display difference frames. For that, select two videos of your playlist, right click on them and choose 'Add Difference Sequence'. The new item shows you the pixel wise difference of the two attached sequences.

If you want to perform a visual test, where participants are asked to rate a set of sequences, text fields in between the sequences come in handy. Just right click on the playlist and select 'Add Text Frame'. You can customize these to your liking and save them alongside your playlist.

Statistics

In YUView, you can add a statistics overlay to your video. This could be data generated by an encoder/decoder (block size, prediction modes, motion information) or other information generated by a separate process. For this purpose we defined a ".csv" file format that carries information (values or vectors) on a block basis. These files can be loaded in YUView and dropped onto a video sequence. YUView then renders the values from the csv file as colored blocks or vectors over the video sequence.

For example if you have a video decoder of some sort, you can write a function that writes out a value per block. This information can then be quickly visualized in YUView. A definition of the csv file can be found in our wiki.

File Options

You have small frame sizes and/or 4k sequences? No problem. YUView has no built in limit on the frame size. And even if you have a non standard frame size, you can customize your frame width and height just like the frame rate via several boxes.

YUView supports various pixel formats like 4:0:0 8-bit, 4:2:0 Y'CbCr 8-bit planar, 4:2:2 Y'CrCb 8-bit planar, 4:4:4 Y'CbCr 12-bit LE planar, RGB 8-bit and GBR 12-bit planar just to name a few. A complete list of all supported formats can be found in our wiki.

Display Options

Are you comparing two video sequences and are you looking for a better way then switching between the two sequences? YUView offers the option to display two videos either in split view or in comparison view. In split view, the two videos are shown in full scale side by side. You can navigate and zoom into both videos just like you could before. In comparison view, both videos are laid out on top of each other, with the view divided by a vertical bar. Right of this bar you can see the one video, on the left, the other video is displayed so you can spot the differences easier. The vertical bar can be moved to the left or right, so you can compare different frame parts of these videos. Simply select two videos in your playlist and set the checkbox "Split- and Comparison View". Now choose either 'comparison' or 'split view' from the dropdown menu.

If you work with decoders or pixel manipulators, you want to know what happens in each block or pixel of your frames. For this, YUView has two features to assist you: a pixel grid and a zoom box. The pixel grid overlays your frame with horizontal and vertical lines in a user defined spacing. The zoom box makes it even more exact. If activated, you can scroll over each pixel of your frame and the zoom box will show you a highly magnified version of the selected pixel and it's surrouding. It will also tell you the exact pixel coordinates and the corresponding pixel Y, U and V values. If you are using statistics , the values of all visible statics will be listed.

YUV Options

YUView is highly flexible when reading YUV files. In this window you can modify the used scales for luma and chroma values and offsets. You can also specify which components to display and which interpolation and color conversion method to use.

Under the hood

Analyze H.265/HEVC streams

Thanks to the libde265 library, YUView can directly decode and display H.265/HEVC encoded binary bitstreams. We even get decoding information directly from the decoder and generate statistics which you can directly view in YUView. Ever wanted to see what is happening when decoding an H.265/HEVC bitstream? Just open it, select a statistic and navigate the sequence. More information can be found in our wiki
